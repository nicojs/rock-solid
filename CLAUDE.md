## What is RockSolid

RockSolid is a management system ("steenvast en solide management systeem") — a Dutch-language web app for managing persons (deelnemers/overige personen), projects (cursussen/vakanties), organisations, locations, and registrations (aanmeldingen).

## Monorepo Structure

This is an npm workspaces + Lerna monorepo with three packages:

- **`packages/shared`** — TypeScript types, REST route definitions, serialization utilities shared between backend and frontend. No build step (consumed as source).
- **`packages/backend`** — NestJS REST API with Prisma ORM (SQLite via `better-sqlite3`). Serves the frontend as static files.
- **`packages/frontend`** — Lit (Web Components) + RxJS frontend, built with Webpack.

## Commands

Run from the **root** of the repo unless noted otherwise:

```bash
# Full build (prisma client → tsc → webpack)
npm run build

# Development mode (tsc watch + backend dev + frontend webpack watch, all concurrent)
npm run dev

# Run all tests
npm test

# Run backend tests only (from root)
npm run test -w packages/backend

# Run a single backend test file (must build first)
cd packages/backend && npm test -- --grep "PersonenController"

# Lint
npm run lint

# Clean all build artifacts
npm run clean

# Build only TypeScript (no frontend)
npm run build:tsc

# Regenerate Prisma client (required after schema changes)
npm run build:db

# Create a new Prisma migration
cd packages/backend && npm run prisma:migrate:create -- <migration-name>

# Reset and push schema to dev database (destructive)
cd packages/backend && npm run prisma:push:force
```

## Architecture

### Backend

- **NestJS** app with controllers, injectable mapper services, and Prisma DB service.
- All API routes are prefixed with `/api`. Static frontend is served from the same process.
- **Mappers** (`packages/backend/src/services/*.mapper.ts`) follow the Data Mapper pattern — they translate between Prisma DB types and shared domain types. Each entity has a dedicated mapper (e.g. `PersoonMapper`, `ProjectMapper`).
- **DBService** (`src/services/db.service.ts`) extends `PrismaClient` and is injected into all mappers.
- **Auth**: JWT guard applied globally via `APP_GUARD`. Office365 OAuth2 strategy for login. `PrivilegesGuard` enforces role-based access (roles: `admin`, `projectverantwoordelijke`).
- **Pipes** (`src/pipes/`) parse and validate query string parameters.
- Generated Prisma client lives in `packages/backend/src/generated/` (gitignored, regenerated by `npm run build:db`).

### Frontend

- **Lit** web components with **RxJS** for reactive state.
- No Angular, no React — all components extend `RockElement` (which extends `LitElement`) and manage subscriptions via a `Subscription` from RxJS.
- Custom client-side **Router** (`src/app/router.ts`) using `History API` and `BehaviorSubject`.
- **RestClient** (`src/app/shared/rest-client.ts`) is a typed HTTP client keyed on `RestRoutes` from `@rock-solid/shared`.
- Feature areas are organized into "front-modules" (e.g. `personen.front-module.ts`) which are plain import files that register components.

### Shared

- **`RestRoutes`** type in `src/rest-routes.ts` is the contract between frontend and backend — defines entity, upsertable entity, and filter types per route.
- **Serialization**: JSON uses a custom `rockReviver` (for `Decimal`, `Date`, etc.) and `parse` function. Both the backend and the test harness apply this reviver.
- Domain types use Dutch naming (Persoon, Aanmelding, Deelnemer, Activiteit, Locatie, etc.).

### Testing

- Tests are **integration tests** using Mocha + Chai + Supertest against a real NestJS app with a real SQLite database (`prisma/test.db`).
- The shared test harness is in `packages/backend/src/test-utils.test.ts`. It bootstraps the full NestJS app, wipes the DB in `beforeEach`, and exposes typed helpers (`harness.createDeelnemer(...)`, etc.).
- Test files are colocated with source: `*.test.ts` (compiled to `dist/**/*.test.js`, picked up by `.mocharc.cjs`).
- Mutation testing uses Stryker (`npm run test:mutation`).

## Code Conventions

- TypeScript with `"type": "module"` (ESM). All backend imports use `.js` extension even for `.ts` source files.
- Prettier: single quotes, trailing commas.
- Domain language is Dutch; code identifiers (variable names, method names, etc.) follow Dutch naming when referring to domain concepts.
